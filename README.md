# lb3_comp

# Лабораторная работа №4: Реализация алгоритма поиска подстрок с помощью регулярных выражений

## Цель
Изучить теоретические основы регулярных выражений и их применение для поиска и извлечения подстрок из текста. Освоить практические навыки использования библиотечных средств работы с регулярными выражениями, а также интеграцию алгоритмов поиска в графический интерфейс приложения.

## Постановка задачи
Разработать модуль поиска подстрок с использованием регулярных выражений, интегрировать его в существующее приложение (текстовый редактор) и обеспечить наглядный вывод результатов.

Студент получает индивидуальный вариант, содержащий 3 задачи на поиск подстрок определенных форматов.

## Задание 1
Построить РВ, описывающее шестнадцатеричный код (HEX) цвета (длиной 3).

### Регулярное выражение:

#([0-9A-Fa-f]{3})\b


### Где:
| Символ | Описание |
|--------|----------|
| `#` | символ решётки в начале HEX-кода |
| `[` | открытие набора символов |
| `]` | закрытие набора символов |
| `0-9` | все цифры от 0 до 9 |
| `A-F` | все заглавные буквы от A до F |
| `a-f` | все строчные буквы от a до f |
| `{3}` | ровно 3 символа из указанного набора |
| `\b` | граница слова (чтобы не захватывать более длинные коды) |

### Верные примеры:
- `#FFF` (белый цвет)
- `#F00` (красный цвет)
- `#0F0` (зелёный цвет)
- `#00F` (синий цвет)
- `#FFA` (желтоватый цвет)
- `#f0f` (розовый цвет, строчные буквы)
- `#123` (серый цвет, цифры)

### Неверные примеры:
- `#FFFF` (4 символа, а нужно ровно 3)
- `#G00` (буква G не входит в шестнадцатеричный диапазон A-F)
- `FFF` (отсутствует символ # в начале)
- `#F2` (только 2 символа, а нужно 3)
- `#F2G` (буква G недопустима)
- `##FFF` (лишний символ #)

## Задание 2
Построить РВ, описывающее номера карт, принадлежащих платежной системе Visa.

### Регулярное выражение:

\b4[0-9]{12}(?:[0-9]{3})?\b


### Где:
| Символ | Описание |
|--------|----------|
| `\b` | граница слова |
| `4` | первая цифра всегда 4 (идентификатор платёжной системы Visa) |
| `[` | открытие набора символов |
| `]` | закрытие набора символов |
| `0-9` | все цифры от 0 до 9 |
| `{12}` | ровно 12 цифр |
| `(?:...)` | незапоминаемая группа |
| `[0-9]{3}` | ровно 3 цифры |
| `?` | предыдущий элемент (группа из 3 цифр) может отсутствовать или присутствовать |
| `\b` | граница слова |

Итого: длина номера карты может быть **13 цифр** (4 + 9) или **16 цифр** (4 + 12).

### Верные примеры:
- `4111111111111111` (16 цифр, начинается с 4)
- `4123456789012` (13 цифр, начинается с 4)
- `4222222222222` (13 цифр)
- `4012888888881881` (16 цифр)
- `4532015112830366` (16 цифр)


### Неверные примеры:
- `5123456789012345` (начинается с 5, это MasterCard, не Visa)
- `411111111111` (только 12 цифр, не хватает 1 цифры для 13-значного номера)
- `411111111111111` (15 цифр, не подходит ни под 13, ни под 16)
- `4111 1111 1111 1111` (содержит пробелы, регулярное выражение ищет только цифры подряд)
- `41111111111111111` (17 цифр, слишком много)
- `3111111111111111` (начинается с 3, это American Express, не Visa)

## Задание 3
Построить РВ для проверки даты, учитывая високосные годы. Формат даты: YYYY-MM-DD.

### Регулярное выражение:

\b(?<year>\d{4})-(?<month>0[1-9]|1[0-2])-(?<day>0[1-9]|[12]\d|3[01])\b


### Где:
| Символ | Описание |
|--------|----------|
| `\b` | граница слова |
| `(?<year>\d{4})` | группа "year" - ровно 4 цифры |
| `-` | разделитель между годом и месяцем |
| `(?<month>0[1-9]\|1[0-2])` | месяц: 01-09 или 10-12 |
| `-` | разделитель между месяцем и днём |
| `(?<day>0[1-9]\|[12]\d\|3[01])` | день: 01-09, 10-29, 30-31 |
| `\b` | граница слова |

### Дополнительная проверка в коде:
После нахождения даты по регулярному выражению выполняется программная проверка:

```csharp
bool isLeapYear = (year % 4 == 0 && year % 100 != 0) || (year % 400 == 0);

if (month == 2)
{
    if (isLeapYear)
        return day <= 29;
    else
        return day <= 28;
}
else
{
    int[] daysInMonth = { 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31 };
    return day <= daysInMonth[month - 1];
}
```

Верные примеры:
2024-02-29 (високосный год, 29 февраля допустимо)

2023-02-28 (не високосный год, 28 февраля допустимо)

2020-02-29 (високосный год)

2023-01-15 (обычная дата)

2024-12-25 (обычная дата)

2023-04-30 (в апреле 30 дней)

Неверные примеры:
2023-02-29 (не високосный год, февраля не может быть 29)

2021-04-31 (в апреле только 30 дней)

2024-13-01 (неверный месяц, месяца 13 не существует)

2024-00-01 (неверный месяц, месяц не может быть 00)

2024-02-30 (в феврале максимум 29 дней)

2023-02-29 (29 февраля только в високосные годы)
